信息分发服务

通过信息分发服务，客户可以将信息传递给同频道的其他客户。

如下图：

1. 客户向分发服务器(频道服务器)发送：

DATA TEXT FING #DISPATCH MESSAGE#大家好！

服务器接收到该消息后将消息转换成：

DATA TEXT FING #SERVER MESSAGE#大家好！

并把它存储到迟后分发缓冲区(dispatchLater)，当分发监测器监测到服务器空闲时便把最前面一条迟后分发命令发布出去，如图中蓝色箭头所示。

2. 客户如果请求分发脚本，如：

DATA TEXT FING #DISPATCH SCRIPT#MSGBOX 123

服务器将把它转换成：

DATA TEXT SCRP MSGBOX 123

这是因为SCRP在传递到客户时客户并不会马上执行该命令，而是等到网络连接结束后才执行。这样就避免了网络堵塞。
