



K studio

M-SC: secret control


- 类型
	- 病毒
		附属代码片断，传染，隐蔽，[自发发作]

	- 特洛伊木马
		完整的包装程序，隐蔽，受控

	- 显示控制
		完整的程序，非隐蔽，受控

	- 后门程序
		附属模块，针对性，隐蔽，受控


- 功能件
	- 代码嵌入与分离
		嵌入：附属代码嵌入宿主程序的方法，有选择的包括加密、编码等
		分离：从宿主程序中分离执行体，分离过程可能是一次性，也可能是逐步的，
		      可能包括解密、解码等。
		需要对宿主程序的格式作分析，以便成功完成嵌入与分离。
	- 感染
		感染必须要求一种先驻留的方法，无论是单线程模型还是多线程模型都有必要。
		因此完成真正感染的部分与驻留是密不可分的。
	- 判断与发作
		可以根据时间等本地信息判断发作条件，也可能受控与网络系统。
		发作也可以是预定义的过程或受控于网络。
	- 自毁
		在必要的时候清理自身的行为。
		可能要求在系统中维护一张列表，以便完全清除，最后清除驻留体。

-
	-
	- 反处理
		反反汇编
		反跟踪，反调试
		反杀毒，通过增强感染算法，使清除变得复杂，等。
